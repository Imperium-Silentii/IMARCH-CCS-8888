# ИНТЕГРАЦИОННЫЙ МОСТ GITHUB-TELEGRAM

## 1. НАЗНАЧЕНИЕ
Обеспечение мгновенной прозрачности. Каждый раз, когда «Акт Реновации» или «Акт Инцидента» попадает в этот репозиторий, система активирует уведомление в официальном Telegram-канале Трастовой Службы.

## 2. ПОТОК ДАННЫХ
1. **Триггер:** `git commit` + `git push` в основную ветку.
2. **Вебхук:** GitHub отправляет данные (JSON) на сервер моста.
3. **Парсинг:** Мост извлекает ID Акта и хеш транзакции.
4. **Трансляция:** Сообщение отправляется в авторизованную группу/канал Telegram.

## 3. АУДИТОРСКИЙ СЛЕД
Этот мост гарантирует, что никто не сможет «тихо» отредактировать Хронику. Любое изменение анонсируется стейкхолдерам в режиме реального времени.
